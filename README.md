# 戦国戦況図タイムラインエディタ

Sengoku Battle Map Timeline Editor は、戦国時代の合戦・城攻め・勢力移動を地図上に配置し、動画編集ソフトのような秒数タイムラインで編集・再生するためのWebアプリです。YouTube歴史解説動画の戦況図素材作成を想定したMVPです。

## 技術スタック

- React
- TypeScript
- Vite
- React Konva / Konva
- Zustand
- 通常CSS
- JSONベース保存

外部APIやサーバーは使っていません。

## 起動方法

```bash
npm install
npm run dev
```

表示されたローカルURLをブラウザで開きます。

## ビルド方法

```bash
npm run build
```

ビルド成果物は `dist/` に出力されます。

## 基本操作

- サンプル選択: 上部ツールバーの「サンプル」から読み込み
- 地図画像読み込み: 「地図画像」ボタンから画像ファイルを選択
- JSON保存: 「JSON保存」または `Ctrl + S`
- JSON読み込み: 「JSON読込」から保存済みJSONを選択
- PNG出力: 「PNG」ボタン
- Undo / Redo: ボタンまたは `Ctrl + Z` / `Ctrl + Y`
- コマ移動: 軍勢コマをドラッグすると現在時間のキーフレームを作成・更新
- 時間移動: 下部タイムラインのスライダー、または `←` / `→` で前後のキーフレームへ移動
- 再生: 下部タイムラインの「再生」
- 線・矢印描画: 描画モードでキャンバスをクリックして点を追加、`Enter` で確定、`Esc` でキャンセル
- パン: `Space + ドラッグ`
- ズーム: マウスホイール
- 削除: 選択中に `Delete`

## JSON保存仕様

プロジェクトは `ProjectData` としてJSON保存されます。主な構成は以下です。

- `timeline`: 秒数キーフレーム、現在時間、補間モード
- `map`: 地図画像Data URL、出力サイズ
- `factions`: 陣営
- `units`: 軍勢コマと秒数キーフレーム
- `sites`: 城・砦・寺・港などの拠点
- `lines`: 包囲線・防御線など
- `arrows`: 進軍・撤退・援軍ルート
- `events`: イベントマーカー
- `labels`: 注釈ラベル

Web版では地図画像をData URLとしてJSON内に保持できます。

## 相対座標

軍勢、拠点、線、矢印、イベント、ラベルの座標は、地図画像上の相対座標で保存します。

```json
{ "x": 0.5, "y": 0.5 }
```

これは地図中央を意味します。絶対ピクセル座標は保存せず、キャンバス表示時だけ内部基準サイズに変換します。

## 動画時間と和暦変換

現在のタイムラインは動画編集用の秒数ベースです。`time` は `"1"`、`"10"`、`"00:10.0"` のような秒数またはタイムコードとして扱います。正確な旧暦・和暦変換は未実装です。

## Tauri化の状況

現時点ではWeb版MVPとして実装しています。Tauri v2は未導入です。ファイル入出力は `src/utils/fileIO.ts` に分離しているため、後からTauriのファイル保存・読み込みAPIへ置き換えやすい構成にしています。
